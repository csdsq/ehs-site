# 法规内容质量标准

> 制定日期：2026-06-12
> 依据：全量诊断118条法规记录，定位并修复8条脏数据记录的经验总结

---

## 一、核心规范

### 1. content 字段规范

**格式：结构化 HTML**

| 场景 | 标签规范 |
|------|---------|
| 法规/文件标题 | `<h3>` |
| 章节标题（章/节/条） | `<h4>` |
| 正文段落 | `<p>` |
| 强调内容（如总则、目录等） | `<p><strong>...</strong></p>` |
| 列表（条/项） | `<p>第X条 ...</p>`，每条独立段落 |

**禁止内容：**
- ❌ 源站原始 HTML（script、style、注释、面包屑导航）
- ❌ JavaScript 代码片段（如 `$('#share').share(...)`）
- ❌ 索引号、文号、发文机构等元信息（这些应放在 description/source 字段）
- ❌ 正文重复（同一段文字出现多次）
- ❌ 导航元素（"当前位置：首页 > 法规文件 > ..."）
- ❌ 无意义换行和空格（由源站换行导致的内容碎片化）

**content 中的 HTML 必须是从原文提取的**，不能是采集工具直接从页面 DOM 序列化的结果。需经过提取正文容器 → 去除脏数据 → 结构化格式化的流程。

### 2. description 字段规范

**原则：撰写实质性的人读摘要**

格式：`{发文机构}发布了《{法规名称}》，{核心内容概述}。`

例如：
- ✅ "全国人民代表大会常务委员会发布了《中华人民共和国危险化学品安全法》，规范危险化学品生产、储存、使用、经营和运输的安全管理，强化全过程安全管控。"
- ✅ "浙江省发展改革委发布了《浙江省绿色低碳技术推广目录（2025年版）》，明确了推广绿色低碳技术的总体要求和具体技术清单。"
- ❌ "首页 > 法规文件 > 当前页"（面包屑）
- ❌ "索引号：XXXXX 发文日期：XXXX"（元信息拼接）

### 3. source 字段规范

**原则：实际发文机关/机构名称**

- ❌ 统一标注为 "国务院" 或 "中国政府网"
- ✅ 省发改委发的 → "浙江省发展改革委"
- ✅ 全国人大发的 → "全国人民代表大会常务委员会"
- ✅ 国家能源局发的 → "国家能源局"
- ✅ 省政府发的 → 具体省/市政府名称

## 二、新法规采集清洗流程

每条法规采集入库后，必须执行以下清洗步骤：

```
1. 提取正文容器
   - 定位页面内的主要内容区域（如 .article-content、#content、.main 等）
   - 排除页头、页脚、侧栏、导航

2. 去除脏数据
   - 移除 <script> 及其内容
   - 移除 <style> 及其内容
   - 移除 HTML 注释
   - 移除面包屑导航（"当前位置：首页 > ..."）
   - 移除页面底部相关链接、分享组件
   - 移除索引号、发文日期等元信息（如果已单独存储）

3. 结构化格式化
   - 将标题转为 <h3>/<h4>
   - 将段落转为 <p>，去除多余换行和空格
   - 每条法条/条文独立成段
   - 章节标题加 <strong> 或 <h4> 标记

4. 内容去重
   - 检查正文是否有重复段落（同一段文字出现多次），保留一次
   - 检查 content 中是否存在 description/source 中的元信息

5. 独立生成 description
   - 基于法规实际内容撰写摘要
   - 包含：发文机构 + 法规名称 + 主要内容概述
   - 不要从页面顶部元信息直接拼接

6. 校验 source
   - 确认 source 等于实际发文机关
   - 浙江省级文件不能标为 "国务院"
   - 部委规章不能标为 "中国政府网"

7. 原文存档（可选）
   - 如采集时从原文 URL 获取，可将原始 URL 存入 original_source_url 字段
   - 方便后续回查和重新提取
```

## 三、常见问题与修复方案

| 症状 | 根因 | 修复方法 |
|------|------|---------|
| 内容包含面包屑导航 | 采集器直接从页面 DOM 获取，未定位正文容器 | 定位正文容器，去除导航元素 |
| 内容含 JS 代码片段 | 采集器序列化整个 body | 移除所有 `<script>` 标签 |
| 正文重复 2-4 遍 | 源站异步渲染导致内容被多次追加到容器 | 内容去重，保留唯一副本 |
| 无段落换行、一切挤在一起 | 采集的 HTML 是压缩后的、或 inline 样式造成 | 结构化格式化，加 `<p>` 分段 |
| 与原文内容不一致 | 采集到的是动态加载的占位内容 | 评估使用原文真实 URL，或等待页面完全渲染后采集 |
| source 标注错误（省文件标为国务院） | 采集时取了页面顶部的"来源：中国政府网"而非实际发文机关 | 根据文件署名或编号规则确定实际发文机关 |
| description 是面包屑拼接 | 直接用了页面顶部元信息 | 人工撰写摘要 |

## 四、定期巡检建议

- 每月或每季度随机抽样 10% 的法规记录检查 content 质量
- 重点检查 URL 以 `.shtml`、`.html` 结尾的源站抓取记录（多为旧式 CMS）
- 新增采集时启用自动校验：content 中是否含 `script`、`$(`、`当前位置` 等关键词
- 发现一条脏数据时，排查同批次采集的其他记录是否同类问题

## 五、审核清单

提交法规内容变更前逐项检查：

- [ ] content 不含 script / style / 注释
- [ ] content 不含面包屑导航
- [ ] content 不含正文重复
- [ ] content 段落结构正确（h3/h4/p 合理使用）
- [ ] description 是人读摘要，不是元信息拼接
- [ ] source 是实际发文机关
- [ ] 原文来源 URL 可访问（如有）
